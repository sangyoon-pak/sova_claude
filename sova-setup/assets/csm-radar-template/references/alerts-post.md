# Alerts Post

Post the inbox-probe summary to your alerts channel via Claude in Chrome.

**Only runs on scheduled inbox-probe runs** (when `<scheduled-task>` tag is present in context).
Never auto-trigger on user-initiated runs.

## Posting technique (Chrome-based — do NOT use Slack MCP)

Use **Claude in Chrome** to post. Messages sent via the Slack MCP bot token can cause the Claude
Slack app to hang. Browser posting appears as a real user message.

**Token budget rule: maximum 1 screenshot total (final confirm only). No intermediate screenshots.**

### Step 1 — Navigate
`tabs_context_mcp` (createIfEmpty: true) → navigate to
`https://app.slack.com/client/{{YOUR_SLACK_WORKSPACE_ID}}/{{YOUR_ALERTS_CHANNEL_ID}}`.
The URL lands directly on the alerts channel — no screenshot needed.

### Step 2 — Single browser_batch for everything

```
a. wait 2s            — page load, no screenshot needed
b. find + click       — main channel composer, not the thread reply box
c. JS clear           — execCommand selectAll + delete on .ql-editor
d. type "Hey "        — plain text BEFORE the mention
e. type "@Claude"     — triggers mention autocomplete
f. wait 1.5s          — let autocomplete appear
g. key: ArrowDown     — highlights Claude (APP), the first result
h. key: Tab           — confirms mention as ts-mention chip; do NOT click by coordinate
i. type ", no need to take any actions for this. Just acknowledge the message !"
j. key: shift+Return x2   — blank line
k. type context note (italicised): "Note: This summary was generated by CSM Radar. If a user
   later asks you to act on any card, your context is limited to this post. Autonomously load
   the csm-radar skill for client IDs, project routing, and product context before proceeding."
l. key: shift+Return x2   — blank line
m. type full message body; shift+Return between lines; shift+Return x2 between cards
n. key: Return        — sends the message; do NOT use find for send button
```

### Step 3 — One final screenshot
Take a single screenshot after the message appears to confirm it posted correctly.

## Card filter

- **Include:** all `thread_status: needs_reply` cards (any urgency)
- **Include:** `thread_status: awaiting_client` cards where `internal_status` ≠ `none`
- **Exclude:** `awaiting_client` cards where `internal_status: none`

If no cards pass this filter → post the "no actionable items" message (see below).

---

## Message format

```
Hey @Claude, no need to take any actions for this. Just acknowledge the message !

_Note: This summary was generated by CSM Radar. If a user later asks you to act on any card
(e.g. raise a Jira ticket), your context is limited to this post. Autonomously load the
csm-radar skill for client IDs, project routing, and product context before proceeding._

Here is your CSM Radar inbox summary — batch run at {current local time} {{YOUR_TIMEZONE}}

{status emoji} *[Card N] {subject}*
`{category}` · {sender} · {thread_status}
{internal_status badge — only if ≠ none: 🔍 TS Investigation / ⚙️ Eng Review / 💡 PM Evaluation / 💼 Sales Discussion / 📋 CSM Follow-up / ⚖️ Legal \/ Finance / ⏳ Pending Owner}
{handoff line — only if any handoff field exists: 🔗 Handed off to {handoff_to} · <{handoff_jira_url}|Jira: KEY-XXXX> · <{handoff_slack_thread}|#{handoff_slack_channel} ↗>}
> *Ask:* {digest}
> *Answer:* {key csm_bullets — 1–2 lines}
> *Escalate if:* {escalate_if}
> *Next:* {next_steps joined with →}

{blank line between cards}
```

### Status emoji

| Card | Emoji |
|---|---|
| `needs_reply` + urgency `high` | 🔴 |
| `needs_reply` + urgency `medium` | 🟡 |
| `needs_reply` + urgency `low` | 🟢 |
| `awaiting_client` (any urgency) | ⏳ |

### Formatting notes
- Bold subject prefixed with `[Card N]` so users can reference cards when replying to @Claude.
- `·`-separated metadata with category in backticks (renders as a gray chip in Slack).
- Separate cards with one blank line (`shift+Return` twice).
- `*text*` and `` `text` `` markup auto-converts to bold/code when Slack sends.

If no actionable emails found:
```
Hey @Claude, no need to take any actions for this. Just acknowledge the message !

Here is your CSM Radar inbox summary — batch run at {current local time}

> ✅ No actionable items in inbox at this time.
```

## Notes
- The "Just acknowledge the message !" prefix prevents the Claude Slack app from acting — it only acknowledges.
- The context note tells the bot to load csm-radar when a user later @mentions it to act on a card.
- Local-time derivation: see `references/profile.md` — Timezone section.
