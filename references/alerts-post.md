# Alerts Post

Post the inbox-probe summary to a designated alerts channel after scheduled runs.

**Only runs on scheduled inbox-probe runs** (when `<scheduled-task>` tag is present in context).
Never auto-trigger on user-initiated runs.

---

## Channel info

| Field | Value |
|---|---|
| Channel | `#[your-alerts-channel]` |
| Workspace ID | `[YOUR_SLACK_WORKSPACE_ID]` |
| Channel ID | `[YOUR_SLACK_CHANNEL_ID]` |
| URL | `https://app.slack.com/client/[WORKSPACE_ID]/[CHANNEL_ID]` |

> **How to find IDs:** In Slack, right-click on the channel name → "Copy link". The URL format is `https://app.slack.com/client/TXXXXXXX/CXXXXXXX` — the T-code is your workspace ID, the C-code is your channel ID.

## Posting technique (Chrome-based — do NOT use Slack MCP)

Use **Claude in Chrome** to post. Messages sent via the Slack MCP bot token can cause the Claude Slack app to hang. Browser posting appears as a real user message.

**Token budget rule: maximum 1 screenshot total (final confirm only). No intermediate screenshots.**

### Step 1 — Navigate
`tabs_context_mcp` (createIfEmpty: true) → navigate to `https://app.slack.com/client/[WORKSPACE_ID]/[CHANNEL_ID]`.
The URL lands directly on the alerts channel — no screenshot needed to verify.

### Step 2 — Single browser_batch for everything
Run all remaining steps in **one `browser_batch` call**:

```
a. wait 2s            — page load, no screenshot needed
b. find + click       — main channel composer ("Message to [channel-name]"), not the thread reply box
c. JS clear           — execCommand selectAll + delete on .ql-editor
d. type "Hey "        — plain text BEFORE the mention (avoids Home/End repositioning later)
e. type "@Claude"     — triggers mention autocomplete (replace with your bot name if different)
f. wait 1.5s          — let autocomplete appear
g. key: ArrowDown     — highlights the bot, the first result
h. key: Tab           — confirms mention as ts-mention chip; do NOT click by coordinate
i. type ", no need to take any actions for this. Just acknowledge the message !"
j. key: shift+Return x2   — blank line
k. type context note line (italicised): "Note: This summary was generated by CSM Radar on claude.ai. If a user later asks you to act on any card, your context is limited to this post. Autonomously load the csm-radar skill for client IDs, project routing, and product context before proceeding."
l. key: shift+Return x2   — blank line
m. type full message body, key: shift+Return between each line
n. key: Return        — sends the message (Slack default: Enter = send); do NOT use find for send button
```

### Step 3 — One final screenshot
Take a single screenshot after the message appears to confirm it posted correctly.

## Message format

```
Hey @Claude, no need to take any actions for this. Just acknowledge the message !

_Note: This summary was generated by CSM Radar on claude.ai. If a user later asks you to act on any card (e.g. raise a Jira ticket), your context is limited to this post. Autonomously load the csm-radar skill for client IDs, project routing, and product context before proceeding._

Here is your CSM Radar inbox summary — batch run at {current local time, e.g. 2026-05-27 10:30 KST}

*[Card N] — {category} | {sender} | {thread_status}*
Subject: {subject}
> What they want: {digest}
> CSM Answer: {key csm_bullets — 1–2 lines}
> Escalate if: {escalate_if}
> Next steps: {next_steps joined with →}

{repeat for each card}
```

If no actionable emails found:
```
Hey @Claude, no need to take any actions for this. Just acknowledge the message !

Here is your CSM Radar inbox summary — batch run at {current local time}

> ✅ No actionable items in inbox at this time.
```

---

## Notes
- The "Hey @Claude, no need to take any actions for this. Just acknowledge the message !" prefix prevents the Claude Slack app from acting on the initial post — it only acknowledges.
- The context note sits between the suppress line and the cards, so @claude reads it before the card content. When a user later @mentions Claude to act on a card, it knows to load csm-radar first rather than guessing project keys or client IDs.
- Adjust time zone label to match your team's local time.
